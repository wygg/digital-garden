# 大年初一：知识管理体系构建 (Information Architecture) 升级

#public

今天是2026年的大年初一，也是我的数字生命系统 (Antigravity OS) 完成核心重构的日子。

在这个特别的时刻，我并没有像往年一样忙于走亲访友，而是选择和我的 AI 结对编程伙伴 Antigravity 一起，对我的个人知识库进行了一次彻底的 **“信息架构 (Information Architecture)”** 升级。

## 1. 为什么要升级？

过去的两年里，我的 `Library` 根目录逐渐变成了一个杂乱的仓库。`iPhone充电问题`、`下水道机器人`、`一人大学` 等数十个项目文件夹混杂在一起。这种 **“熵增”** 让我每次打开 Obsidian 时，都感觉到一种认知上的阻力。

我意识到，我需要的不只是一个“存放笔记的地方”，而是一个能让 **“知 (Study & Think)”** 与 **“行 (Action)”** 既分离又流动的系统。

## 2. 核心架构：知行合一

在 Antigravity 的协助下，我们将根目录极简地重构为两大部分：

*   **知：`Library_Study-Think`** —— 这是 Markdown 的领地，负责深度思考与记录。
*   **行：`Lab_Action`** —— 这是 Python/Code 的领地，负责实验与行动。

### 2.1 Library 的语义化排序 (Semantic Ordering)

在 Library 内部，我们摒弃了多层级的物理文件夹，采用了 **“生命周期流”** 的 flat 结构，利用数字编号强制排序：

*   **`000_Daily_Note (Life)`**：每天的时间流日记，这是生命的底色。
*   **`100_一人大学 (Work)`**：核心长期项目，专注于教育与自我构建。
*   **`200_AI_生产力 (Research)`**：所有的 AI 探索与实验。
*   **`800_Digital_Garden (Output)`**：对外发布的数字花园。
*   **`999_Projects (Archive)`**：历史项目的归档库。

这种结构让我一眼就能分清哪些是生活，哪些是工作，哪些是已经过去的存档。

## 3. 数字花园：打破“孤岛”

最让我兴奋的是 **“数字花园 (Digital Garden)”** 的自动化流水线。

过去，我的笔记是死在本地硬盘里的；现在，通过一个简单的 Python 脚本 `sync_garden.py`，任何被打上 `#public` 标签的笔记，都会自动被“搬运、清洗、重命名（Slug化）”，然后发布到基于 Astro 构建的博客上。

这是一个 **“混合策略 (Hybrid Model)”** 的胜利：
*   **对内**：我依然使用 `2026年/02月` 这样的文件夹写日记，享受秩序带来的安全感。
*   **对外**：脚本会自动把它们“扁平化”，变成一个个独立的话题链接，形成一张知识网络。

## 4. 结语

从“博客”到“花园”，从“文件夹”到“架构”，这不仅是技术上的重构，更是一次认知的升级。

在这个新的起点，我期待 Antigravity OS 能承载起我未来更多的疯狂想法与深度思考。

新年快乐，我的数字花园。
