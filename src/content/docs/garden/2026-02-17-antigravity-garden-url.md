# 🗂️ Antigravity Garden 知识管理体系与 URL 策略

**版本**：1.0
**日期**：2026-02-17
**核心哲学**：**内部有序（Order）** vs **对外网络（Network）** 的平衡。

---

## 1. 核心矛盾分析 (The Architect vs. The Gardener)

在个人知识管理 (PKM) 中，存在两个对立的需求：
*   **存档需求 (Archiving)**：像传统图书馆一样，按时间严格入库，方便检索。
*   **展示需求 (Presenting)**：像维基百科一样，按主题相互连接，方便漫游。

## 2. 解决方案：混合策略 (Hybrid Model)

采用 **“物理层级存储 + 逻辑扁平发布”** 的双层结构。

### 2.1 输入端 (Obsidian) —— 物理层级
保持您现有的 **年/月/日** 文件夹结构，提供极其稳定的心理安全感。

```text
Library/Daily_Note/
├── 2026年学思笔记/        【一级：年份容器】
│   ├── 2026年02月/        【二级：月份归档】
│   │   ├── 2026-02-17 关于春节的思考.md
│   │   └── ...
```

**优点**：
*   **低认知负担**：每天只要打开当月文件夹，不用思考“放在哪个主题下”。
*   **可扩展性**：无论一年写几百篇，目录永远清晰。

### 2.2 处理端 (Sync Script) —— 逻辑映射
使用 Python 脚本进行“降维打击”：

1.  **递归扫描**：脚本深入所有子文件夹，找出带 `#public` 标签的笔记。
2.  **扁平化搬运**：无视原来的层级，统一提取到 Digital Garden 的根目录。
3.  **重命名魔法**：将文件名转换为 URL 友好的格式（见下节 Slug）。

### 2.3 输出端 (Astro Garden) —— 网络展示
在最终网页上，不再展示文件夹，而是展示：
*   **时间流 (Timeline)**：按 `date` 字段自动排序。
*   **知识图谱 (Graph)**：按双向链接关系展示。
*   **话题索引 (Tags)**：按 `tags` 聚类。

---

## 3. URL 策略详解 —— The Slug

**定义**：Slug 是 URL 中唯一标识文章的短字符串（如 `my-first-post`）。

### 3.1 为什么重要 (Why Slug Matters)
*   **可读性**：`wygg.com/garden/digital-garden-manifesto` 比一串乱码的中文字符更有意义。
*   **稳定性**：中文 URL 在复制粘贴时容易变长、截断或变成乱码 `%E4%B8%80...`。
*   **SEO**：搜索引擎更喜欢语义化的英文 URL。

### 3.2 生成规则 (Generation Rules)

脚本在同步时将遵循以下优先级：

1.  **手动指定 (最佳)**：
    *   在 Obsidian 笔记的 Frontmatter 中包含 `slug: my-custom-url`。
    *   **适用场景**：重要的项目文档、SOP、深度长文（如“一人大学计划”）。
    
2.  **自动转化 (默认)**：
    *   如果未指定 Slug，脚本自动将中文标题转换为 **拼音** 或 **英文翻译** (未来接入 LLM)。
    *   **适用场景**：日常随笔、碎片化思考。
    
    *示例转换*：
    *   原标题：`2026-02-17 关于数字花园的思考.md`
    *   Slug：`2026-02-17-thoughts-on-digital-garden`

## 4. 行动指南 (Action Items)

2.  **打标签**：想发布的笔记，打上 `#public`。
3.  **加 Slug**：对于非常得意的文章，手动在开头加上 `slug: your-english-title`。
4.  **交给脚本**：剩下的搬运、重命名工作，全权交给 `sync_garden.py`。
